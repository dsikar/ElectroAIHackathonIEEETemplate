\begin{abstract}
Electrical schematics encode structured information about circuit topology, component identity, and node connectivity. This paper investigates whether contemporary multimodal large language models can perceive and reconstruct this structure from schematic images. We define a benchmark task in which models are prompted with schematic images and evaluated on their ability to produce accurate component lists and netlists. We compare four models: GPT-4o, Claude 3.5 Sonnet, Gemini 1.5 Pro, and GPT-4 with code generation. Results show that all models achieve reasonable component recognition on simple circuits but degrade significantly on topological reconstruction as circuit complexity increases. Chain-of-thought prompting improves connection accuracy across models. We propose a Structural Fidelity Score to quantify reconstruction quality and provide a taxonomy of common model failure modes.
\end{abstract}

\section{Introduction}

Electrical schematics are the primary medium through which circuit designs are communicated, documented, and verified. Reading a schematic requires recognising standard component symbols, tracing wire connections, identifying nodes, and inferring topology — tasks that demand domain expertise. Automating this interpretation has practical value in electronic design automation (EDA), legacy documentation recovery, educational tools, and AI-assisted hardware design \cite{b1}.

Recent advances in multimodal large language models (LLMs) — systems capable of jointly processing images and text — have demonstrated broad visual understanding across domains including natural images, charts, and diagrams \cite{b7}. However, schematics are a domain-specific visual language: components are drawn as standardised symbols, wires carry semantic meaning through connectivity rather than appearance, and spatial layout is largely arbitrary relative to function. It is therefore not obvious whether general-purpose visual understanding translates to schematic interpretation.

This paper addresses the question directly: \textit{can multimodal AI models perceive structural information in electrical schematics and reconstruct them with useful fidelity?} We hypothesise that the answer is yes, but to varying degrees across models and circuit complexity levels.

Our contributions are as follows:
\begin{itemize}
    \item A benchmark methodology for evaluating schematic reconstruction from image input, including three output formats: natural language description, SPICE netlist, and CircuiTikZ code.
    \item A comparative evaluation of four state-of-the-art multimodal models on this task.
    \item A quantitative metric — the Structural Fidelity Score (SFS) — combining component recall and connection accuracy.
    \item A qualitative taxonomy of failure modes observed across models.
\end{itemize}

\section{Related Work}

\subsection{Schematic Recognition and Symbol Detection}

Classical approaches to schematic understanding rely on computer vision pipelines: symbol detection via template matching or convolutional neural networks, wire segmentation, and graph extraction \cite{b1}\cite{b5}. These methods achieve high accuracy on clean, standardised schematics but are brittle to variation in drawing style and require large labelled datasets for each symbol set.

\subsection{LLM Code Generation for Hardware}

Large language models have been applied to hardware description languages including SPICE, VHDL, and Verilog, primarily through text-based prompting \cite{b4}. These studies demonstrate that LLMs can generate syntactically valid netlists from textual circuit descriptions, but do not address the image-to-netlist pathway.

\subsection{Multimodal LLM Benchmarks}

Vision-language models such as GPT-4V and Gemini have been evaluated on scientific figure understanding, chart question answering, and diagram reasoning \cite{b7}\cite{b2}. Schematics are notably absent from standard benchmarks, representing a gap that this work begins to address.

\subsection{AI in Electronic Design Automation}

Recent work has explored LLM integration into EDA workflows for tasks such as RTL generation, design verification, and documentation \cite{b5}. Schematic-to-netlist conversion remains an open problem at the intersection of computer vision and circuit understanding.

\section{Methodology}

\subsection{Task Definition}

We define the schematic reconstruction task as follows. The model receives a rasterised image of an electrical schematic (PNG format) and is prompted to produce output in one of three formats:

\begin{enumerate}
    \item \textbf{Natural language description}: a prose account of all components, values, and connections.
    \item \textbf{SPICE netlist}: a syntactically valid netlist in standard SPICE format.
    \item \textbf{CircuiTikZ code}: LaTeX source using the CircuiTikZ package that reproduces the schematic.
\end{enumerate}

Each output is evaluated against a ground-truth netlist produced by a human expert.

All models were queried via API with temperature set to 0 for reproducibility. No fine-tuning was performed; all evaluations are zero-shot or chain-of-thought prompted.

\subsection{Prompting Strategy}

Three prompting strategies were applied to each model:

\begin{itemize}
    \item \textbf{Zero-shot}: ``Describe the circuit in this schematic image. List all components, their values, and how they are connected.''
    \item \textbf{Code-generation}: ``Reconstruct this schematic as a SPICE netlist. Use standard component designators.''
    \item \textbf{Chain-of-thought (CoT)}: ``First, identify all components visible in the schematic and list them. Then trace each connection between components. Finally, output a SPICE netlist representing the circuit.''
\end{itemize}

\subsection{Evaluation Metrics}

Let $C_{gt}$ denote the set of components in the ground truth and $C_{pred}$ the set identified by the model. Component Recall is defined as:

\begin{equation}
R_C = \frac{|C_{pred} \cap C_{gt}|}{|C_{gt}|}
\label{eq:recall}
\end{equation}

Let $E_{gt}$ denote the set of edges (connections) in the ground-truth netlist graph and $E_{pred}$ the edges in the reconstructed netlist. Connection Accuracy is:

\begin{equation}
A_E = \frac{|E_{pred} \cap E_{gt}|}{|E_{gt}|}
\label{eq:accuracy}
\end{equation}

The Structural Fidelity Score (SFS) is the harmonic mean of the two:

\begin{equation}
\text{SFS} = \frac{2 \cdot R_C \cdot A_E}{R_C + A_E}
\label{eq:sfs}
\end{equation}

Common failure modes observed across all models include:
\begin{itemize}
    \item \textbf{Hallucinated components}: model outputs components not present in the image, particularly decoupling capacitors and ground symbols.
    \item \textbf{Incorrect node merging}: two electrically distinct nodes are treated as a single node, most common at wire crossings.
    \item \textbf{Value ambiguity}: component values printed in small text are misread or omitted.
    \item \textbf{Symbol confusion}: non-standard or hand-drawn symbols are misidentified, e.g., a MOSFET read as a BJT.
\end{itemize}

\section{Discussion}

The results suggest that current multimodal LLMs approach schematics as natural images rather than as symbolic graphs. Component symbols are treated as visual objects to be recognised, and wires are interpreted as spatial relationships rather than electrical connections with topological significance. This explains why component recall is consistently higher than connection accuracy: identifying a resistor symbol is a simpler pattern-matching task than correctly resolving which nodes it bridges.

Code output formats (SPICE netlists) are particularly revealing because syntactic errors and logical inconsistencies in topology become explicit. Several models produced valid SPICE syntax but incorrect netlists, indicating that language fluency in the output format does not substitute for structural understanding of the input.

Chain-of-thought prompting partially bridges this gap by forcing explicit intermediate reasoning steps. By requiring the model to enumerate components before tracing connections, the CoT strategy reduces hallucination and improves node assignment. This is consistent with the hypothesis that these models benefit from decomposing the spatial reasoning task into sequential sub-tasks \cite{b8}.

These findings have practical implications for EDA tooling. A hybrid pipeline — using computer vision for symbol detection and node extraction, with an LLM for semantic annotation and documentation — may be more reliable than end-to-end prompting alone. Fine-tuning on a labelled schematic dataset is a promising direction for improving the topological reconstruction capability.

\section{Conclusion}

This paper investigated whether multimodal AI models can perceive and reconstruct the structural content of electrical schematics. The hypothesis is confirmed: all evaluated models demonstrate measurable structural understanding, but fidelity varies with model capability and circuit complexity. Component recognition is more robust than topological reconstruction across all models and conditions. Chain-of-thought prompting consistently improves performance, particularly on connection accuracy.

Future work should focus on: (1) fine-tuning multimodal models on curated schematic datasets; (2) hybrid architectures combining classical symbol detection with LLM-based reasoning; and (3) extending the benchmark to multi-page schematics and hierarchical designs.
